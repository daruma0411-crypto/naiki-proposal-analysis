# NAIKI向け提案分析レポート

## エグゼクティブサマリー

ナイキ株式会社（オフィス家具メーカー）の営業プロセスを分析し、デジタル化による効率化の提案をまとめた。特に「製品選定から図面作成」の工程に大きな改善余地があることが判明。

---

## 🔍 重要なファクト

### ファクト1: 営業プロセスのボトルネック

**現状**: ヒアリング→図面作成→提案のサイクルに時間がかかり、修正を3～5回繰り返す

#### 📊 プロセスの詳細分解

```
【第1サイクル】初回提案まで（約3週間）
├─ Day 1-2: ヒアリング（2日）
│   └─ 課題: その場で概算見積もりが出せない
│
├─ Day 3-5: 製品選定（3日）
│   └─ 課題: 70万点のDBから選定、属人的
│
├─ Day 6-10: 図面作成（5日）
│   └─ 課題: デザイナー不足で待ち時間発生
│
├─ Day 11-14: 3Dパース作成（4日）
│   └─ 課題: 外注コスト高、時間かかる
│
└─ Day 15-18: 見積書作成（4日）
    └─ 課題: 手作業、転記ミス

【第2サイクル以降】修正対応
├─ フィードバック受領
├─ 修正作業（5日）
└─ 再提案（2日）
→ これを3～5回繰り返す
```

#### ⚠️ 主要ボトルネック

| ボトルネック | 所要時間 | 原因 |
|-------------|----------|------|
| 製品選定の属人化 | 9～15日 | 70万点から経験と勘で選ぶ |
| 図面・3Dパース作成待ち | 27～45日 | デザイナー不足、外注待ち |
| 見積書の手作業 | 12～20日 | 手入力、転記ミス |
| 修正サイクルの多さ | 3～5回 | その場で代替案提示不可 |

## 🛒 商品選定の5つの困りごと

### 1. 膨大な製品から探す作業
- **70万点**のDBから探す
- 検索軸が多い（サイズ、価格、色、材質、機能、納期...）
- **例**: 「20名、予算300万円、ナチュラル」→ **3時間以上**

### 2. 在庫・納期確認の待ち時間
```
営業が問い合わせ → 生産部門が確認（待ち）
→ 回答（数時間～1日） → 「在庫なし」
→ 振り出しに戻る
```

### 3. 価格計算の煩雑さ
- 定価 → 顧客ランク割引 → キャンペーン → まとめ買い → 配送費
- 製品ごとに価格体系が異なる
- 手計算でミスのリスク

### 4. 最適な組み合わせの判断

**カタログ調査から判明した重要な発見**:
```
基本: デスク5種 × チェア5種 = 25パターン

さらに複雑化:
+ ついたて/パーティション: 10種類以上
  - 高さ（ロー/ハイ）
  - 素材（布/アクリル/スチール）
  - 色・デザイン
+ 収納、配線、天板色、脚タイプ...

→ 実質的に組み合わせは無限
```

**セット販売でさえカスタマイズ余地が大きい**:
- 「〇〇シリーズのオフィスセット」として売られていても
- **ついたてを変えるだけで印象が大きく変わる**
- 顧客は「基本はこれだけど、ここだけ変えたい」と言う

**ビジュアルイメージが重要**:
- ついたての有無や高さでオープン感が変わる
- 色の組み合わせで雰囲気が全く異なる
- **カタログの静的な写真だけでは顧客がイメージできない**

### 5. 曖昧な要望の解釈
- 「おしゃれ」→ モダン？北欧風？
- 「機能的」→ 収納力？可動性？
- 顧客自身も明確でない

## 💡 デジタルツールの必要性

### 従来（カタログ）の限界
```
「Aセット」「Bセット」の写真を見せる
↓
「ついたてはこんな種類があります」と別ページ
↓
顧客は頭の中で組み合わせを想像するしかない
```

### デジタルツールで実現できること
```
「Aセット」を表示
↓
ついたてを選択 → リアルタイムで見た目が変わる
↓
高さを変える → 印象の変化を確認
↓
色を変える → 雰囲気の違いを比較
↓
その場で「これだ！」と決められる
```

## 📐 簡易図面作成に必要な情報とツール

### 必要な情報

#### オフィス情報
- **必須**: 幅・奥行き・天井高（mm単位）
- **便利**: 壁・柱・窓・ドア・コンセント位置

#### 製品情報
- **必須**: 製品名、サイズ、数量
- **便利**: 形状データ、重量、配線

#### 制約条件
- 動線（通路1m以上）
- 部署配置
- 視線・プライバシー配慮

### 必要なツール機能

#### A. 間取り作成
- 矩形の部屋作成、壁・柱配置
- グリッド表示、スナップ機能

#### B. 製品配置
- ドラッグ&ドロップ、回転
- 等間隔配置、壁沿い配置
- **セット一括配置**

#### C. 組み合わせ試行（重要！）
- **ついたて/パーティションの付け替え**
- 天板色・チェアの入れ替え
- **リアルタイムでビジュアル更新**

#### D. 2D表示・出力
- 平面図、寸法線
- PDF/DXF出力

#### E. 見積もり計算
- 自動計算、割引適用
- **オプション変更時の差額表示**

## 🎯 具体的な提案：プレレイアウトツールMVP

### Phase 1: 最小機能（50～80万円、2ヶ月）

#### 機能
1. **間取り入力**
   - 矩形の部屋（幅・奥行き入力）
   - 壁の配置

2. **製品選定サポート**
   - 条件検索（予算、サイズ、カテゴリ）
   - **セット商品の選択**
   - 製品リスト表示

3. **製品配置**
   - ドラッグ&ドロップ
   - 回転、移動、削除
   - **セット一括配置**

4. **組み合わせ試行機能（カタログ調査から判明した必須機能）**
   - **ついたて/パーティションの付け替え**
     - 高さ選択（ロー/ハイ）
     - 素材選択（布/アクリル/スチール）
     - 色選択
   - 天板色の変更
   - チェアの入れ替え
   - **変更するとビジュアルが更新される**

5. **2D表示**
   - 平面図表示
   - 寸法表示
   - ついたての表現（高さで色分け）

6. **見積もり自動計算**
   - 製品単価×数量
   - 合計金額表示
   - **オプション変更時の差額表示**

7. **出力**
   - PDF出力（平面図+製品リスト+見積）
   - Excel出力

#### 使用技術
- フロントエンド: React + Konva.js（2D描画）
- バックエンド: Node.js + Express
- DB: PostgreSQL（製品データ、セットマスタ、組み合わせマスタ）

#### この機能で解決される課題
- 「ついたてを変えたらどう見えるか」→ **その場で確認**
- 「セット + カスタマイズ」→ **ベースはセット、細部は調整**
- 「組み合わせ無限問題」→ **顧客が自分で試せる**

## まとめ

### 核心的な発見

1. **営業プロセスの可視化**: 製品選定→図面作成の間に大きな断絶
2. **組み合わせ無限問題**: セット販売でも、ついたて等の変更で印象が激変
3. **ビジュアル化の重要性**: カタログの静止画では限界、リアルタイム試行が必須
4. **デジタルツールの価値**: 「セット + カスタマイズ」を両立し、意思決定を加速

### 差別化ポイント

**大手**: 高度なAI、複雑なシステム
**NAIKI**: **シンプルで使いやすい、組み合わせが自由に試せる実用ツール**

中小企業向け「100人オフィス戦略」にマッチした、親しみやすいデジタル体験を提供。
